## 2. Data Contract — `khatira_content.json`

**Generated by**: `dart run tool/generate_elm_lists.dart --json`

**Path**: `lib/features/khatira/data/static/khatira_content.json`

### JSON Schema

```json
{
  "version": 1,
  "generated_at": "2026-06-21",
  "chapters": [
    {
      "id": "pre",
      "order_index": 0,
      "title": "المقدمة",
      "pages": [
        {
          "index": 0,
          "titles": ["عنوان الصفحة الأولى"],
          "subtitles": ["عنوان فرعي"],
          "texts": ["نص المحتوى"],
          "ayahs": ["آية قرآنية"],
          "footer": "تذييل الصفحة",
          "order": ["titles", "subtitles", "texts", "ayahs", "footer"]
        }
      ]
    }
  ]
}
```

### Field Semantics

| Field | Type | Description | Display |
|-------|------|-------------|---------|
| `titles` | `string[]` | Page titles (0–2) | Title/headline (larger font) |
| `subtitles` | `string[]` | Sub-section headings (0–2) | Subtitle (medium font) |
| `texts` | `string[]` | Main body content (1–6) | Body text |
| `ayahs` | `string[]` | Quranic verses / hadith (0–3) | Distinct style (italic/quotes) |
| `footer` | `string` or null | Footnote / reference | Small text at bottom |
| `order` | `string[]` | Display order of the above fields | Determines layout sequence |

### Chapter ID Mapping

| ID | Meaning | Route |
|----|---------|-------|
| `pre` | Introduction | 0 |
| `1`–`32` | Ders 1–32 | 1–32 |
| `final` | Final chapter | 33 |

### Important Notes

- **Arabic text only** — RTL layout required throughout
- **All strings use Arabic script** — no translations exist
- **Font family**: Amiri (Arabic naskh style) — use Amiri on Android too
- **Some fields can be null/empty** — handle gracefully
- **`order` array dictates display sequence** — render fields in this exact order


## 8. Critical Gotchas

1. **RTL is REQUIRED** — Arabic text throughout. Set `android:supportsRtl="true"` in manifest. Use `LayoutDirection.Rtl` in Compose.
2. **Amiri font** must be bundled — not available on all Android devices. Add `.ttf` to `res/font/`.
3. **JSON file is ~500KB** — load it once on startup via `AssetManager`, cache in memory. Don't re-read on every screen.
4. **`order` field is CRITICAL** — the display sequence varies per page. Do NOT hardcode a fixed order. Always render fields in the `order` array sequence.
5. **Screen rotation** — ViewModel survives, but `HorizontalPager` position must be saved/restored via `savedStateHandle`.
6. **No Flutter-specific features needed** — avoid translating Flutter paradigm directly; use idiomatic Android patterns (e.g., don't recreate Bloc, use ViewModel + StateFlow).
